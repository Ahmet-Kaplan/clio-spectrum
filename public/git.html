Using Git/Github with CLIO

Stuart Marquis, Programmer, LITO



What is Source Code Control?
  • File backups
  • File backups, by date
  • Backups of complete directories
  • Backups of complete directories to a shared server
What is Git?
  • Really good source code control
  • Useful features:
    • Fast, Simple
    • Checkout, Edit, Commit
    • Commit History
    • Version Comparison
    • Branching
    • Push / Pull
    • Lots of commands
    • A few more
  • Clients for every platform
What is GitHub?
  • Shared server for Git projects
  • Works for any size project - Solo or Team, Simple or Complex
  • Used by anybody - individual developers or organizations
  • Used by Columbia Libraries for a number of projects
  • Used by a number of other University Libraries
  • How do they do it?
CLIO uses GitHub
  • Developers use Git
  • All Source Code is on GitHub
  • CLIO releases are GitHub releases
  • CLIO issues become GitHub commits
  • CLIO is Deployed from GitHub
  • Some Benefits: Control, Safety, Tracking, Reliability, Nifty Graphs
